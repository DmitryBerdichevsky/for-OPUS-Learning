Версии проекта

В течение жизненного цикла проекта разрабатываемая в рамках проекта конфигурация постоянно развивается- при этом требуется разработка и выпуск новых версий проекта (конфигурации). В СППР для проекта ведется учет версий, данные версии используются при проектировании и разработке, исправлении ошибок, планировании и т.д.

Для версии можно указать адрес хранилища, в котором ведется разработка (т.ч. можно указать, что версия разрабатывается в основном хранилище). Эта информация будет использоваться при загрузке метаданных и обработчиков обновления информационной базы.

Для версии, для которой указано, что она разрабатывается в отдельном хранилище, можно указать отличия в именах объектов метаданных и составе подсистем по сравнению с основным хранилищем. Эта информация используется при генерации текстов описания обработчиков обновления версии, поэтому указание отличий доступно в проектах, в которых включено ведение списка обработчиков обновления.
